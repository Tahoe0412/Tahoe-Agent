# 用户手册

版本：`0.1.0`  
适用对象：品牌方、运营团队、内容团队、创作者  
产品定位：品牌营销策略、内容生产与运营自动化一体化工作台

## 1. 这是什么

这套平台不是单次生成工具，而是一套从“想法”走到“可发布内容”的工作台。  
当前主要支持三类工作模式：

- `短视频`
- `文案写作`
- `宣传推广`

核心原则：

- 先把项目目标说清楚
- 再收集数据与研究趋势
- 再生成内容初稿
- 最后做改写、合规和优化

## 2. 建议你先怎么用

如果你是第一次使用，建议按下面顺序：

1. 新建项目
2. 选择工作模式
3. 填写项目介绍、核心想法、原始输入
4. 如果是文案类项目，再设置：
   - 写作模式
   - 输出风格
   - 文案长度
   - 使用场景
   - 风格参照 / 参考样稿
5. 跑一版结果
6. 再进入对应工作台细修

## 3. 主要页面说明

### 3.1 总览

入口：`/`

作用：

- 查看当前项目状态
- 快速进入下一步
- 启动新项目

你应该在这里做什么：

- 看当前项目是否已经选好方向
- 按页面推荐的“下一步”进入对应模块

### 3.2 创意任务单

入口：`/brief-studio`

作用：

- 把本轮任务说清楚
- 定义目标、核心表达和 CTA

默认只保留最重要的字段，方便快速启动。  
如果只是先跑通项目，不必把所有高级约束一次写满。

### 3.3 趋势研究

入口：`/trend-explorer`

作用：

- 查看值得做的主题
- 判断本轮优先方向

当前页面默认只突出：

- `优先做` 1 个主题
- `备选` 2 个主题

其余趋势默认折叠，避免信息过载。

### 3.4 脚本实验台

入口：`/script-lab`

作用：

- 查看原始文本
- 修改 AI 重构文本
- 重跑分类和素材分析

默认建议你只做三件事：

1. 选一个镜头
2. 改 `AI 重构文本`
3. 点保存或重跑

高级字段已经收起来，只有需要时再展开。

### 3.5 分镜编排

入口：`/scene-planner`

作用：

- 看当前镜头是否能开工
- 看缺什么素材
- 上传素材

默认先看：

- 当前镜头
- 素材缺口
- 上传入口
- 快速重跑按钮

### 3.6 宣传文案与运营

入口：`/marketing-ops`

这是当前文案类项目最重要的页面。

作用：

- 生成宣传主稿
- 生成平台稿
- 做合规检查
- 做主稿诊断与增强

推荐流程：

1. 先选：
   - 写作模式
   - 输出风格
   - 文案长度
   - 使用场景
2. 确认项目里已经填写：
   - 项目介绍
   - 核心想法
   - 原始输入
   - 风格参照 / 参考样稿
3. 点击 `生成宣传主稿`
4. 如果结果一般，点击 `诊断并增强主稿`
5. 满意后再生成平台稿
6. 最后做合规检查

### 3.7 品牌档案

入口：`/brand-profiles`

作用：

- 保存品牌基础资料
- 沉淀品牌 voice
- 定义禁用表达
- 配置内容支柱

建议：

- 一个品牌建一个档案
- 不要把项目级信息写进品牌档案

### 3.8 行业模板

入口：`/industry-templates`

作用：

- 沉淀行业关键词
- 管理竞品关键词
- 定义风险词和表达边界

建议：

- 一个行业建一个模板
- 让多个项目复用同一套边界

### 3.9 项目中心

入口：`/project-hub`

作用：

- 管理多个项目
- 搜索、筛选、置顶、归档
- 批量绑定品牌和行业模板

推荐做法：

- 正在推进的项目：`置顶`
- 暂时不做的项目：`归档`
- 不建议直接删除历史项目

## 4. 如何创建一个高质量文案项目

如果你想先跑通“收集数据 -> 分析 -> 写出完整宣传文案”的流程，建议这样做：

1. 新建项目  
   模式选：
   - `文案写作`
   - 或 `宣传推广`

2. 在项目里补清楚这几项：
   - 项目介绍
   - 核心想法
   - 原始输入

3. 设置文案控制项：
   - 写作模式
   - 输出风格
   - 文案长度
   - 使用场景

4. 如果你有喜欢的参考文风：
   - 把 1 到 3 段样稿贴到 `风格参照 / 参考样稿`

5. 先去趋势研究页看方向

6. 再去宣传文案与运营页生成主稿

7. 如果第一版不够好：
   - 点 `诊断并增强主稿`

8. 主稿满意后：
   - 再生成平台稿
   - 再做合规检查

## 5. 如何提高生成质量

最重要的不是多点几次，而是把输入写清楚。

优先做好这几件事：

- `项目介绍`
  - 写清楚你是谁、面向谁、现在做什么

- `核心想法`
  - 一句话写清本轮最重要的表达

- `原始输入`
  - 尽量提供真实需求、原始脚本或核心材料

- `风格参照 / 参考样稿`
  - 建议贴 2 到 3 段完整文案
  - 不要只贴几个标题
  - 最好是同一类风格，不要风格混杂

- `写作模式`
  - 决定“写什么”

- `输出风格`
  - 决定“怎么写”

- `文案长度`
  - 决定展开程度

- `使用场景`
  - 决定结构和转化方式

## 6. 模型使用建议

当前平台支持按步骤路由模型，不同环节可以用不同模型。

建议思路：

- 趋势分析：用更稳的分析型模型
- 宣传主稿：用更擅长中文传播表达的模型
- 平台改写：用更擅长结构改写的模型
- 报告总结：用更稳的收束型模型

如果你看到结果质量突然明显下降，先检查：

- 当前是否真的调用了模型，而不是 mock
- 项目输入是否太少
- 写作模式和使用场景是否选错

## 7. 多项目管理建议

推荐用法：

- `置顶`
  - 当前重点项目

- `标签`
  - 按季度、活动、品牌线、产品线管理

- `归档`
  - 暂时停掉的项目

不建议一开始就做“硬删除”，因为项目里沉淀的是资产：

- 趋势结论
- 任务单
- 脚本
- 分镜
- 平台稿
- 合规记录

## 8. 常见问题

### 为什么生成结果像摘要，不像宣传文案？

常见原因：

- 项目介绍太空
- 核心想法不清楚
- 使用场景选错
- 没有提供风格参照
- 当前用的是 mock 或不合适的模型

建议：

- 补项目初始信息
- 再点 `诊断并增强主稿`

### 为什么我找不到某个模块？

系统会根据你选的项目模式收缩导航。  
不同模式下，左侧模块不完全一样。

### 为什么不建议直接删除项目？

因为项目不只是一个标题，而是一整套可复用资产。  
建议先 `归档`。

## 9. 文档更新规则

这份手册建议每个版本至少更新这 3 部分：

1. `主要页面说明`
2. `推荐流程`
3. `常见问题`

如果某个版本新增或调整：

- 页面入口
- 核心流程
- 模型配置方式
- 项目管理方式
